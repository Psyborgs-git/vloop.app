## 2024-03-12 - [Terminal] Limit try-catch scope in input validation hot path
**Learning:** In V8 engines, wrapping a whole execution block (like looping over regex tests) inside a `try-catch` can prevent compiler optimizations, severely hindering performance on frequently called functions (hot paths) like payload input validation for terminals.
**Action:** When working on very hot paths where inputs stream at high frequency, narrow the scope of `try-catch` blocks specifically around the code that actually needs to catch expected exceptions (e.g. `new RegExp(pattern)` for invalid patterns), keeping the caching mechanism (`Map.get`) and fast-path execution (`re.test`) strictly outside the `try-catch` block.
## 2025-01-20 - [Web UI] Custom areEqual for React.memo in streaming components
**Learning:** Components that receive unstabilized callback props (e.g. arrow functions re-created on every render cycle of a parent like `useChat`) will bypass `React.memo` entirely because shallow prop comparison fails on those new function references. This causes O(N^2) re-rendering of entire chat histories during streaming as new chunks arrive.
**Action:** When using `React.memo` on list items in a streaming interface (like `ChatMessageItem`), explicitly provide a custom `areEqual` function that compares ONLY the relevant data props (e.g. `prevProps.msg.content === nextProps.msg.content`), explicitly ignoring unstabilized callback props.
