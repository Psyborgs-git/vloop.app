## 2024-05-18 - [Web UI React Iteration Anti-Pattern]
**Learning:** React components displaying collections of data (like logs, executions, or histories) in this codebase are highly susceptible to performance degradation from polling (`setInterval` state updates) triggering O(N) filtering operations synchronously in the render flow. For lists with potentially unbound growth like `WorkflowRunsView` executions, multi-pass filtering combined with auto-polling creates a direct correlation between list size and main thread blockage during UI updates.
**Action:** When updating lists or grids that use continuous polling, consolidate map/filter logic into single-pass loops and wrap derived computations spanning the full list in `useMemo` hooks. Ensure auto-polling components aggressively decouple expensive state derivation from the main render path.
